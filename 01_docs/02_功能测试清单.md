## ⚠️ 安全提示

### API Key 安全使用指南

#### ✅ 正确做法

1. **使用环境变量**
   ```bash
   # Linux/Mac
   export OPENAI_API_KEY="your-key-here"
   export OPENAI_BASE_URL="https://api.deepseek.com/v1"
   export OPENAI_MODEL="deepseek-chat"

   # Windows (CMD)
   set OPENAI_API_KEY=your-key-here
   set OPENAI_BASE_URL=https://api.deepseek.com/v1
   set OPENAI_MODEL=deepseek-chat

   # Windows (PowerShell)
   $env:OPENAI_API_KEY="your-key-here"
   $env:OPENAI_BASE_URL="https://api.deepseek.com/v1"
   $env:OPENAI_MODEL="deepseek-chat"
   ```

2. **使用配置文件**
   ```bash
   # 创建或编辑 ~/.qoocode/config.json
   {
     "apiKey": "your-key-here",
     "baseURL": "https://api.deepseek.com/v1",
     "model": "deepseek-chat"
   }
   ```

#### ❌ 禁止做法

1. **不要将 API Key 写入代码文件**
   ```typescript
   // ❌ 错误
   const apiKey = "sk-xxx"

   // ✅ 正确
   const apiKey = process.env.OPENAI_API_KEY
   ```

2. **不要在测试脚本中硬编码 API Key**
3. **不要在文档中暴露真实 API Key**
4. **不要将 API Key 提交到 Git**

#### 🔒 安全最佳实践

1. **定期轮换 API Key** (每 30-90 天)
2. **使用权限受限的 Key**
3. **监控 API 使用情况**
4. **保护配置文件** (`chmod 600 ~/.qoocode/config.json`)

#### 📋 提交前检查清单

- [ ] 没有硬编码的 API Key
- [ ] 没有 API Key 在测试脚本中
- [ ] 没有 API Key 在文档中
- [ ] .env 文件在 .gitignore 中
- [ ] 配置文件不在版本控制中
- [ ] 使用 `git grep "sk-"` 检查是否有敏感信息

## 📋 最佳实践应用

### 测试原则

1. **独立测试**: 每个测试独立运行，不依赖其他测试
2. **清晰断言**: 使用明确的期望和实际值
3. **边界测试**: 测试空值、特殊字符等边界情况
4. **错误处理**: 验证错误路径和恢复机制
5. **快速反馈**: 单元测试运行速度快，集成测试覆盖真实场景

describe('File Operations', () => {
